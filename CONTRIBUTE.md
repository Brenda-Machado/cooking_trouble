# Guia de Contribuição

Obrigado por querer contribuir com o **Cooking Trouble**.  
Este projeto é de código aberto e todas as contribuições são bem-vindas, sejam correções, novas funcionalidades, melhorias de código, documentação ou design.

---

## Como Contribuir

Siga os passos abaixo para enviar suas contribuições de forma organizada:

1. **Faça um fork** deste repositório.
2. **Clone o fork** para o seu ambiente local:
   ```bash
   git clone https://github.com/seu-usuario/cooking-trouble.git
   ```
3. Crie uma nova branch para a sua contribuição:
```bash
   git checkout -b minha-feature
   ```
4. Implemente suas alterações e faça commits descritivos:
    ```bash
   git commit -m "Adiciona nova funcionalidade X"
   ```
5. Envie as alterações para o seu fork:
   ```bash
   git push origin minha-feature
   ```
6. Abra um pull request! Descreva claramente as mudanças feitas e o motivo delas.

## Tipos de Contribuição

As principais formas de contribuir são:

+ Correção de bugs: Identifique e corrija problemas no código.

+ Melhorias de código: Refatore e aplique boas práticas de programação.

+ Novas funcionalidades: Adicione novas mecânicas, sistemas ou elementos de gameplay.

+ Design e arte: Melhore sprites, cenários e elementos visuais.

+ Documentação: Atualize ou melhore a clareza dos arquivos de documentação.


## Boas Práticas

Antes de enviar suas contribuições, siga estas boas práticas:

+ Use mensagens de commit curtas e objetivas.

+ Teste suas alterações antes de enviar o Pull Request.

+ Mantenha o código limpo, organizado e consistente com o estilo existente.

+ Respeite as diretrizes de formatação e organização de pastas do projeto.

+ Seja colaborativo e mantenha a comunicação clara nas discussões e revisões.
